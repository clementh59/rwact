Zero configuration. The user doesn't select protocols, doesn't set thresholds. We figure out what matters based on what they actually own. Now watch what happens when something changes...
Here's the login flow. The mobile app sends a POST /register to our Express server, which handles auth and stores data in SQLite. On register or when a new address is added, the matching engine kicks in: it fetches portfolio data from DeBank, processes it to compute leverage and exposure, then matches tokens with the right monitoring workflows.
The workflow management layer handles all the monitoring logic. Earnings calls are tracked via Alpha Vantage with reminders at 7 days and 1 day before. Dividends are detected on-chain by subscribing to the MultiplierUpdated EVM event, then computing the dividend per share. xStocks points are polled from the xStocks API. SEC filings are tracked via EDGAR. And for Twitter monitoring, we use the Otomato SDK to subscribe to relevant accounts, run tweets through an AI relevance filter, and push notifications only when they matter to the user's holdings.
The execution layer. User actions and automated strategies go through a simplified API with three endpoints: /trade, /earn, and /leverage coming soon. Under the hood, we abstract complexity by routing through Cowswap and Tydro. Everything runs on an ERC-4337 smart account deployed on Ethereum via ZeroDev's battle-tested kernel. This gives us two key things: secure automation via session keys and call policies, and better UX with social accounts and batching.
